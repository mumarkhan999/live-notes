Instance methods and Class methods

Instance methods: Work with instance variables For example: setters and getters
Class methods: Work with class variables

class Student:
 
    school = 'Junior Campus High School'
    
    # instance method
    def __init__(self, marks1, marks2):
        self.marks1 = marks1
        self.marks2 = marks2
 
    # instance method
    def avg(self):
        return (self.marks1 + self.marks2) / 2
 
    @classmethod
    def get_school(cls):
        return cls.school
 
 
s1 = Student(10, 20)
s2 = Student(42, 36)
 
print(s1.avg())
print(Student.get_school())
Static methods
  • Instance methods deal with instance variables
  • Class methods deal with class variables
  • Static methods doesn't have to do anything with instance variables or class vairables. Though at variables level static variables and class variables are same but on methods level static methods and class methods are two different things. Static methods has nothing to do with variables. We can do something in static method like interacting with other classes.
class Student:
 
    school = 'Junior Campus High School'
    
    # instance method
    def __init__(self, marks1, marks2):
        self.marks1 = marks1
        self.marks2 = marks2
 
    # instance method
    def avg(self):
        return (self.marks1 + self.marks2) / 2
 
    @classmethod
    def get_school(cls):
        return cls.school
 
    @staticmethod
    def info():
        print('Hi, I am student class.')

s2 = Student(42, 36)
 
print(s1.avg())
print(Student.info())

If we dont' define constructor in subclass it will call the constructor of parent class. But if we define the constructor in subclass, upon making in instance of subclass, it will only call
the constructor of subclass.

Case: sublcass with no constructor

class A:

Case: sublcass with constructor

class A:
 
    def __init__(self):
        print('in A init')
        
    def feature1(self):
        print('feature 1')
 
class B(A):
 
    def __init__(self):
        print('in B init')
        
    def feature2(self):
        print('feature 2')

Output will be

Case: We also want to call the constructor of subclass as well as base class

class A:
 
    def __init__(self):
        print('in A init')
        
    def feature1(self):
        print('feature 1')
 
class B(A):
    
    def __init__(self):
        super().__init__()
        print('in B init')
 
    def feature2(self):
        # we can use super() mtehod to call other methods too not only __init__
        super().feature1()
        print('feature 2')
 
 
b = B()

Output will be

in A init
in B init

Case: Multiple Inheritence, MRO (METHOD RESOLUTION ORDER)
Well just imagine that you have inherited your child class with two different classes but both parent classes have a method with similar name. So on calling that method using the instance of child class which method will be called? It depends on the order of inheritence that you mentioned e.g.: class C(A, B):. Here you wrote A first means coming from left to right so as A comes first so the method from A will be called.

class A:
 
    def feature1(self):
        print('feature 1 from A')
 
class B:
 
    def feature1(self):
        print('feature 1 from B')
 
 
class C(A, B):
 
    def __init__(self):
        print('in C init')
    
 
c = C()
c.feature1()

Output will be

in C init
feature 1 from A

1. Duck Typing

Duck typing in computer programming is an application of the duck test—"If it walks like a duck and it quacks like a duck, then it must be a duck"—to determine if an object can be used for a particular purpose. With normal typing, suitability is determined by an object's type. In duck typing, an object's suitability is determined by the presence of certain methods and properties, rather than the type of the object itself.

Operator Overloading

We can use magic/dunder methods for that

class Student:
 
    def __init__(self, marks):
        self.marks = marks
 
    def __gt__(self, other):
        if self.marks > other.marks:
            return True
        return False
    
    def __lt__(self, other):
        if self.marks < other.marks:
            return True
        return False               
 
    def __str__(self):
        return 'Student has {} marks.'.format(self.marks)
 
s1 = Student(10)
s2 = Student(20)
 
print(s2>s1)
print(s2<s1)

Output will be

True
False

    # initialize answer as 0 
    if datatype =='int': 
        answer = 0
          
    # if datatype is str

    if datatype =='str':

    for x in args:
